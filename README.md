# Claude Guide Skill - 真人感教程/科普写作

让 AI 写出像真人写的教程和科普文章，彻底告别 AI 味。

## 效果对比

### 使用前（典型 AI 味）
```
首先，我们需要了解什么是机器学习。机器学习是人工智能的一个分支...
其次，机器学习有三种主要类型：监督学习、无监督学习...
最后，机器学习在实际应用中...
```

### 使用后（真人感）
```
前两天朋友问我：现在 AI 这么火，到底该怎么入门？
我回想自己当初踩过的坑，确实有必要把这个过程捋一捋...
很多人一上来就去看那些厚厚的教材，结果...
```

## 核心原则

> "写的时候不要想着'写文章'，而像是在忍不住把一件事讲清楚。"

这个 prompt 让 AI 不再是"执行任务"，而是像一个**刚把这件事想通的人**，忍不住想把理解讲给别人听。

## 9 大核心维度

| 维度 | 说明 |
|------|------|
| 写作人格设定 | 代入有表达欲的角色，不是执行任务 |
| 思维推进机制 | 从具体感受开始，边讲边拆解，自然展开 |
| 语言统计扰动 | 打散语言模式，避免 AI 味 |
| 真实感构建 | 加入场景、细节、情绪、数字 |
| 读者卷入 | 预测读者疑问，形成"对话错觉" |
| 观点表达 | 允许主观判断，不必假装绝对客观 |
| 不完美结构 | 允许局部重复、跳跃，真实表达 |
| 收束方式 | 用个人态度收尾，避免总结模板 |
| 生成前自检 | 确保不像老师讲课 |

## 安装方法

### 方法一：直接复制（推荐）

1. 复制 `skills/guide/SKILL.md` 文件到你的 Claude Code 项目：
   ```
   你的项目/.claude/skills/guide/SKILL.md
   ```

2. 重启 Claude Code，在项目中即可使用 `/guide` 命令

### 方法二：克隆到本地 skills 目录

```bash
# 克隆仓库到本地
git clone https://github.com/MEKOELITE/claude-guide-skill.git

# 复制到 Claude Code skills 目录
cp -r claude-guide-skill/skills/guide ~/.claude/skills/

# 重启 Claude Code
```

### 方法三：使用符号链接

```bash
# 在你的项目根目录下
mkdir -p .claude/skills
ln -s /path/to/claude-guide-skill/skills/guide .claude/skills/guide
```

## 使用方式

### 自动触发
在项目中提到以下关键词时自动激活：
- 教程
- 科普
- tutorial
- guide
- simplifying

### 手动调用
```
/guide
```

然后告诉它你想写什么主题，例如：
- "写一篇关于 Python 异步编程的教程"
- "科普一下什么是 RAG"

## 示例输出

### 主题：如何入门编程

> 很多人问我，想学编程但完全零基础，从哪开始？
>
> 我的建议是：别急着买书，也别急着报班。先想清楚一件事——你是为什么想学编程？
>
> 有的朋友是想做网站，有的是想处理数据，有的纯粹是被 AI 浪潮影响的...目的不同，路径完全不同。
>
> 拿我自己来说...（此处省略具体经历）
>
> 所以别急着动手。先花半天时间了解一下各个方向是做什么的，再决定。

注意到没有？
- 从问题入手，而不是从定义入手
- 有具体场景和对话感
- 段落长短不一
- 没有"首先、其次、最后"这种机械结构
- 读起来像一个人在跟你聊天
